# Dives — 投資追蹤 PWA 使用手冊

台股 / 美股個人投資組合追蹤 App。純前端、零後端,**資料完全存在你的裝置本機**,可選擇用 GitHub Gist 跨裝置同步。可「加入主畫面」變成像原生 App 的圖示,**不需要 Apple Developer Program、不會過期**。

🔗 **線上使用:https://weihao0107.github.io/Dives-web/**

## 1. 安裝到手機(啟用)

網址:`https://weihao0107.github.io/Dives-web/`

### 📱 iPhone / iPad(Safari)

1. 用 **Safari**(iPhone 必須用 Safari)開啟上方網址
2. 點畫面**最下方中間的「分享」按鈕**(方框 + 向上箭頭)
3. 往下滑,點「**加入主畫面**」
4. 名稱保留「Dives」,右上角點「**新增**」
5. 主畫面出現 **Dives** 圖示,點開即為全螢幕 App

> ⚠️ **重要**:iOS 的「Safari 分頁」與「主畫面 App」儲存空間是**分開的**。請務必**先加入主畫面,再從圖示打開 App** 來輸入資料 / 設定同步 / 設定密碼,否則資料不會出現在圖示版。

### 📱 Samsung / Android

可用 **Samsung Internet** 或 **Chrome**,擇一即可:

**Samsung Internet**
1. 開啟網址
2. 點下方工具列的**選單(≡ 三條線)**
3. 點「**新增網頁至**」→「**主螢幕**」(部分版本網址列右側會直接出現「安裝 ⤓」圖示)
4. 確認 → 主螢幕出現 **Dives** 圖示

**Chrome**
1. 開啟網址
2. 點右上角**選單(⋮)**
3. 點「**安裝應用程式**」或「**加到主畫面**」→ 安裝
4. 主螢幕(或 App 抽屜)出現 **Dives** 圖示

> Android 安裝後是**獨立全螢幕 App**,會自動更新,體驗與原生 App 接近。
> 生物辨識解鎖在 Android 用的是**指紋 / 臉部解鎖**(原理同 iOS Face ID,見第 8 節)。

## 2. 快速開始

1. 從主畫面圖示打開 App
2.(可選)到「設定 → 起始現金」輸入金額,持倉頁就會顯示現金與淨資產
3. 在「投資組合」頁點右下角 **＋** 新增第一筆交易
4. 點右上角 **⟳** 更新報價

底部四個分頁:**投資組合 / 歷史 / 報表 / 設定**。

## 3. 投資組合

顯示總覽與持倉明細。

- **總覽卡**:總淨資產、今日損益、總損益、報酬率、未實現損益
- **配置條**:台股(橙)/ 美股(藍)佔比,有設定現金時顯示現金餘額
- **篩選**:`全部` / `台股` / `美股`
  - **`全部` 模式會把美股換算成台幣(NT$)**,與台股統一單位方便比較
  - `美股` 分頁則以美元($)顯示
- **排序**:市值 / 損益 / 現價 / 佔比
- **點任一持股** → 查看該檔交易明細,可新增該檔交易或刪除整檔
- 上方總覽固定,**只有下方持倉列表會捲動**

> 顏色慣例:**漲紅、跌綠**(台股慣例)。

## 4. 新增與編輯交易

點投資組合右下角 **＋**(或在持股明細裡新增):

- **買入 / 賣出** 切換
- **股票代碼**:輸入後會自動完成
  - 台股:可用**代號或名稱**搜尋(例:`2330`、`台積電`)
  - 美股:用**代號**搜尋(例:`AAPL`)
- **交易日期**
- **股數 / 價格**
- **手續費**:可選「費率 %」或「固定金額」
  - 預設費率:台股 `0.1425%`、美股 `0.08%`(選股後自動帶入對應預設)
- 下方即時顯示「買入/賣出金額」與「預估手續費」

**編輯 / 刪除**:在「歷史 → 交易紀錄」或持股明細點該筆交易即可編輯;編輯視窗內也可刪除。賣出時會自動計算已實現損益。

## 5. 歷史

上方切換「**趨勢**」與「**交易紀錄**」,**上方控制固定、下方內容捲動**。

- **趨勢**:淨資產堆疊面積圖(台股橙在下、美股藍疊上、相加為總資產)
  - 時間區間:`1M / 3M / 6M / YTD / 1Y / 全部`
  - 點圖表任一處顯示當日台股 / 美股 / 總資產數值
- **交易紀錄**:預設顯示
  - 篩選 `全部 / 台股 / 美股`、可搜尋代號或名稱
  - 點任一筆 → 編輯 / 刪除

## 6. 報表

年度 / 月度的績效彙整(資料來自每日快照)。

- **年度 / 月度** 切換;月度可選年份
- **摘要橫幅**:最新一期的本期損益、累計損益、報酬率、未實現
- **資產走勢圖**:台股 / 美股堆疊面積圖
- **資料表**(預設由舊到新,點「期間」可切換升降序):

  | 欄位 | 說明 |
  |------|------|
  | 期間 | 年份 / 月份 |
  | 淨資產 | 期末淨資產 |
  | 本期投入 | 本期新增投入 |
  | 本期損益 / 報酬率 | 本期損益與本期報酬率 |
  | 未實現 / 已實現 | 期末未實現、本期已實現 |

- **點任一欄位標題**,上方圖表會切換成該欄位的長條圖(再點一次回走勢圖):
  - 淨資產 / 本期投入 → 藍色長條
  - 本期損益 → 正綠、負紅
  - 未實現 / 已實現 → 未實現(虛線框)+ 已實現(實心),台股正紅負綠

## 7. 雲端同步(跨裝置)

資料預設只存單一瀏覽器。要在多裝置共用同一份資料,用 **GitHub Gist 自動同步**。

### 步驟一:產生 GitHub Token

1. 開啟(已預選 `gist` 權限):
   `https://github.com/settings/tokens/new?scopes=gist&description=dives-sync`
2. **Note** 填 `dives-sync`(可改)
3. **Expiration**:想一勞永逸選 `No expiration`;較安全可設一年後
4. 確認 **`gist`** 已勾選
5. 拉到最底 → **Generate token** → 複製那串 `ghp_...`(離開頁面就看不到了)

> 請用 **classic token**(上面連結即是);只勾 `gist` 一個權限,這把鑰匙**只能讀寫你的 Gist**,碰不到程式碼或其他資料。

### 步驟二:在 App 啟用

1. 設定 → **雲端同步** → 貼上 token → **啟用**
2. 第一台有資料的裝置會把資料上傳,建立一個你帳號裡的**私人 Gist**

### 步驟三:其他裝置

- 在其他裝置 / 瀏覽器,貼上**同一組 token** → 啟用 → 自動拉下同一份資料

### 運作方式

- **開啟 App / 回到前景** → 自動拉取(遠端較新則更新)
- **新增 / 編輯 / 匯入 / 清空** → 自動推送(延遲 2 秒)
- **按 ⟳ 重新整理** → 雙向同步 + 更新報價
- 以時間戳判斷新舊,**較新的勝**

> ⚠️ 限制:同一時間在兩台裝置同時編輯,後存的會覆蓋先存的(單人跨裝置正常使用幾乎不會遇到)。Token 只存各裝置本機,不會寫進 Gist。

## 8. App 鎖定(密碼 / Face ID)

把 App 鎖起來,開啟時需驗證,保護裝置上的財務資料。

### 啟用

1. 設定 → **App 鎖定** → **啟用 App 鎖定**
2. 設定 **4–6 位數字密碼**(輸入兩次確認)
3.(裝置支援時)打開 **Face ID / Touch ID** 開關 → 完成生物辨識註冊

### 設定項目

- **變更密碼**
- **Face ID / Touch ID** 開關
- **自動鎖定**:立即 / 1 / 5 / 15 分鐘 / 1 小時(背景超過此時間回來需重新解鎖)
- **停用 App 鎖定**

### 解鎖

- 開啟 App / 逾時回前景 → 顯示鎖定畫面
- 已設定 Face ID → **自動觸發**生物辨識,看一眼即解鎖
- 也可直接輸入 PIN;Face ID 失敗時用 PIN 備援

> ℹ️ 生物辨識:iPhone 用 **Face ID / Touch ID**(需 iOS 16.4 以上、從主畫面圖示開啟);Samsung / Android 用**指紋 / 臉部解鎖**。這是「本機隱私閘門」(無後端),能擋住他人隨手翻看,非銀行等級驗證。
> **忘記密碼**:目前無法重設密碼,需清除瀏覽器網站資料(資料會一併清除),再從 CSV 備份或雲端同步還原。**請務必先備份或啟用同步。**

## 9. 資料備份與還原

設定 → **資料備份**:

- **匯出備份**:下載 `portfolio_backup_YYYY-MM-DD.csv`(含交易紀錄 + 每日快照)
- **匯入備份**:選 CSV 還原(會覆蓋現有資料)
- **重建歷史走勢圖**:用交易紀錄 + 台股歷史收盤,補回過去每一天的資產曲線

> CSV 格式與 iOS 原生 App 相容,可雙向搬移。資料只在本機,**建議定期匯出備份**。
> 走勢圖看的是「每日快照」,只有開過 App 的日子才有。輸入過去日期的交易後,按「**重建歷史走勢圖**」即可補出完整曲線(美股歷史以成本估算)。

## 10. 從 iOS App 搬移資料

1. iOS App → 設定 → **匯出備份** → 取得 CSV
2. 用 AirDrop / Email / 雲端把 CSV 傳到手機
3. PWA(從主畫面圖示開啟)→ 設定 → **匯入備份** → 選該 CSV
4. 交易紀錄與每日快照都會還原

## 12. 常見問題

**Q. 台股價格好像不是最新的?**
盤中會顯示即時價;若在收盤後或非交易時段,顯示的是當日 / 前一交易日收盤,屬正常。按 ⟳ 可手動更新。

**Q. 輸入了過去月份的交易,走勢圖卻沒有那段曲線?**
走勢圖看的是「每日快照」,只有你開過 App 的日子才會記錄。輸入過去日期交易後,到「設定 → 資料備份 → **重建歷史走勢圖**」,系統會用台股歷史收盤回推過去每一天的資產,補出完整曲線(美股歷史段以成本估算)。

**Q. 不同瀏覽器 / 裝置資料不一樣?**
資料預設各瀏覽器獨立。要共用請啟用「雲端同步」(見第 7 節)。

**Q. 同步失敗?**
確認 token 是 **classic** 且勾了 `gist` 權限、未過期。錯誤訊息會顯示在同步狀態列。

**Q. 忘記 App 鎖定密碼?**
需清除瀏覽器該網站資料(資料會清空),再從 CSV 或雲端同步還原 —— 所以請先備份 / 開同步。

**Q. 更新了 App 但畫面沒變?**
把 App 完全關閉(從多工列上滑移除)再重開,Service Worker 會自動載入新版。

**Q. 資料安全嗎?**
資料只存你的裝置本機;啟用同步時存到你**自己 GitHub 帳號的私人 Gist**。開發者與他人都看不到你的資料。
